# Approval Report Template (Gates A/B)

## Capability

Name and identifier.

## Objective

What problem is being solved.

## Current status

Researching, proposed, implemented, tested or waiting for approval.

## Confirmed decisions

Only previously approved decisions.

## Assumptions

Unconfirmed assumptions that currently influence the design.

## Proposed implementation

Concise description of the functionality.

## Engineering methodology

Standards, equations, models and units.

## Architecture impact

Entities, APIs, files, background jobs and integrations.

## Security impact

Permissions, organization isolation and data risks.

## Increment plan

Small implementation steps in recommended order.

## Tests and validation

How correctness will be demonstrated.

## Risks and limitations

Known concerns and excluded scope.

## Acceptance criteria

Objective conditions for approval.

## Decision required

List the exact decisions required from the Coordinator Agent or human product owner.
